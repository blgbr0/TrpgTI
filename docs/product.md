# 产品方案

## 1. 项目目标

`团桌生物鉴定` 的目标不是做科学量表，而是做一个跑团圈层的社交货币。

首版要满足四件事：

1. 用户能在 3 分钟内测完。
2. 用户测完后愿意截图或复制结果发群。
3. 用户会认领结果，或立刻想到自己车队里的某个人。
4. 内容结构足够清晰，后续可以继续扩题、扩人格、扩彩蛋。

## 2. 参考 SBTI 的核心路径

真正要复刻的是传播链路，而不是具体测试内容：

1. 借用“人格测试”外壳，降低理解门槛。
2. 用短题和直觉作答，降低参与负担。
3. 用高梗密度命名，提升第一次结果冲击。
4. 用半冒犯、半认领的文案，提高分享意愿。
5. 用适合截图的结果页，推动群聊裂变。
6. 用四字母代号强化“测试感”，用中文外号强化“点名感”。

## 3. 产品定位

- 品类：泛跑团娱乐型人格测试
- 核心用户：玩过桌面角色扮演游戏的玩家、KP/DM、群内围观党
- 核心场景：群聊转发、开团前团建、复盘后互相认领、朋友圈晒图
- 核心语气：半阴湿、半沙雕、带一点跑团桌边的熟悉狠劲

## 4. 首版体验流

1. 首页看懂“这是个什么测试”。
2. 点击开始后进入 24 题答题流。
3. 每次单题作答，展示当前进度。
4. 完成后生成 Top 3 匹配结果。
5. 结果页展示主人格、典型表现、维度雷达替代信息、可分享文案。
6. 页面底部提供人格图鉴，方便继续传播和围观。

## 5. 内容生产原则

1. 先写人格，再写题。
2. 先写分享文案，再补解释文案。
3. 先保证“看起来像你”，再追求“算法更准”。
4. 所有题目优先使用跑团场景语言，而不是心理学语言。
5. 人格主标签优先使用 `社会绰号 / 职业标签 / 轻污名词`，而不是中性功能名。
6. 结果页文案要像一段可截图传播的外号判词，而不是产品说明书。

## 6. 项目架构

### 文档层

- `docs/product.md`：产品定位、传播逻辑、内容原则
- `docs/meme-bank.md`：跑团高频梗、黑话与可用命名素材
- `docs/sbti-study.md`：SBTI 如何模仿 MBTI，以及跑团版应该如何借壳和变体
- `docs/scoring.md`：题目维度、向量匹配、评分算法
- `docs/roadmap.md`：实施路径、验证指标、后续扩展
- `docs/personas/*.md`：每个人格的单独文档

### 实现层

- `src/data.js`：人格数据、题库、维度定义
- `src/app.js`：答题状态、评分逻辑、渲染
- `src/styles.css`：首版视觉与响应式布局
- `index.html`：静态入口页

## 7. 首版成功标准

1. 用户能不读说明直接开始做题。
2. 结果页能一屏看到最重要的信息。
3. 人格标签具有可截图传播性。
4. 数据结构允许继续加隐藏人格、体系分支和题库版本。
