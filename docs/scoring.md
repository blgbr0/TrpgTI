# 内容与评分架构

## 1. 首版结构

- 主题题目：24 题
- 彩蛋题目：2 至 4 题（2 道随机插入入口题，最多展开 2 道追问题）
- 维度数量：12 维
- 正式人格：16 种
- 隐藏人格：4 种
- 匹配方式：基于 12 维向量的最近邻匹配

## 2. 12 个隐藏维度

1. `PUSH` 推进欲
2. `DIG` 深挖欲
3. `SHOW` 表演欲
4. `TALK` 社交欲
5. `CHAOS` 整活欲
6. `SAFE` 保命欲
7. `RISK` 作死欲
8. `RULE` 规则执念
9. `LORE` 设定投入
10. `PAIN` 沉浸受苦度
11. `SUS` 多疑度
12. `LUCK` 骰运宿命感

每个维度由 2 道题承载，每道题 4 个选项，对应分值 `0-3`。

彩蛋题不参与维度计分，只参与隐藏人格判断。

## 3. 用户画像向量生成

1. 记录每题选择的分值。
2. 按维度汇总该维度下的两道题分值。
3. 取平均值，得到 12 个 `0-3` 的用户维度分。

示例：

```text
Q1 = 3, Q2 = 2
PUSH = (3 + 2) / 2 = 2.5
```

## 4. 人格画像向量

每种人格定义一组固定目标向量，例如：

```text
CLUE = {
  PUSH: 2,
  DIG: 3,
  SHOW: 1,
  TALK: 1,
  CHAOS: 0,
  SAFE: 1,
  RISK: 1,
  RULE: 1,
  LORE: 2,
  PAIN: 1,
  SUS: 2,
  LUCK: 1,
}
```

## 5. 匹配算法

首版使用曼哈顿距离：

```text
distance = sum(abs(user[dim] - type[dim]))
```

最大距离为：

```text
12 个维度 × 每维最大差值 3 = 36
```

匹配度计算：

```text
match = round((1 - distance / 36) * 100)
```

再用两个附加指标做排序修正：

1. `exactMatches`：用户维度四舍五入后与类型目标完全一致的维度数量
2. `distance`：距离越小越优先

最终排序：

1. `distance` 升序
2. `exactMatches` 降序
3. `code` 升序稳定输出

## 6. 题目设计原则

1. 所有题目都必须是场景题。
2. 选项不出现“明显正确答案”。
3. 题面尽量控制在手机一屏内。
4. 选项之间体现程度差，而不是价值判断。
5. 保持 `60% 日常题 + 25% 冲突题 + 15% 发病题` 的整体配比。

## 7. 特殊机制

### 7.1 隐藏人格 `PIGEON 鸽子精`

首版加入了一个直接参考 `SBTI` 思路的彩蛋机制：

1. 答题流中会随机插入 `SP1`。
2. 若用户在 `SP1` 选择“在路上了，马上”，则立即插入 `SP2`。
3. 若用户在 `SP2` 选择偏“还没起床/还没准备好”的选项，则触发隐藏人格 `PIGEON 鸽子精`。
4. 隐藏人格会覆盖标准人格结果，但标准 Top 3 仍然保留在结果页供围观。

这个机制对应的是跑团群内最稳定的社交笑点之一：`开团前的时间魔法`。

### 7.2 隐藏人格 `BLANK 白板怪`

第二个彩蛋机制对应另一个高频桌边笑点：`开团前最后二十分钟才开始车卡`。

1. 答题流中会随机插入 `SP3`。
2. 若用户在 `SP3` 暴露自己其实还没准备好人物卡，则进入 `SP4`。
3. 若用户在 `SP4` 继续暴露“现在才开始点技能”或“打算边开边编”，则触发隐藏人格 `BLANK 白板怪`。
4. 隐藏人格会覆盖标准人格结果，但标准 Top 3 仍然保留在结果页供围观。

这层彩蛋的意义，是把 `SBTI` 的“门控式羞辱”迁移到跑团里最经典的 `卡发了吗` 场景。

### 7.3 隐藏人格 `LOOT 刮地皮`

第三个彩蛋机制对应的是跑团里另一种高频病症：`案发现场第一反应不是破案，而是舔包`。

1. 答题流中会随机插入 `SP5`。
2. 若用户在 `SP5` 暴露出明显的捡东西冲动，则进入 `SP6`。
3. 若用户在 `SP6` 继续承认“先装包里”“没焊地上就是野生资产”，则触发隐藏人格 `LOOT 刮地皮`。
4. 隐藏人格会覆盖标准人格结果，但标准 Top 3 仍然保留在结果页供围观。

这一层把 `SBTI` 的“抽象标签 + 日常羞辱”翻译成跑团场景里的另一种高度共识行为：`比起查案，更着急清空地面资源`。

### 7.4 隐藏人格 `SANO SAN0`

第四个彩蛋机制对应的是 `COC` 玩家非常稳定的一种病症：`明知道会掉san，还是想亲眼看看`。

1. 答题流中会随机插入 `SP7`。
2. 若用户在 `SP7` 暴露出明显的“让我先看看”倾向，则进入 `SP8`。
3. 若用户在 `SP8` 继续承认自己会为了看清楚而硬吃精神伤害，则触发隐藏人格 `SANO SAN0`。
4. 隐藏人格会覆盖标准人格结果，但标准 Top 3 仍然保留在结果页供围观。

这一层把 `掉san` 从一个纯规则术语，翻译成了玩家群体里非常好笑也非常真实的好奇心病症。
